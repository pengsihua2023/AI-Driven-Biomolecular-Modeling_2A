# Draft Data Management and Sharing Plan (for Award Negotiation)

> DOE indicates that a DMSP is not required as a Project Narrative appendix at application time under this Phase I instruction set, but selected applicants should be prepared to provide one during award negotiation. Treat this as a ready-to-finalize draft.

## 1. Overview

This project will generate and curate research data, trained model artifacts, evaluation outputs, workflow code, and supporting metadata related to AI-driven biomolecular modeling of H5N1 antigenic evolution, immune escape, and cross-species adaptation. The team will manage these outputs to maximize reproducibility, appropriate access, and downstream scientific reuse.

## 3. Standards, metadata, and documentation

The team will document data provenance, curation rules, split definitions, model configuration, software versioning, compute environment, and evaluation protocols. Metadata will be structured to support findability and reuse.

### Minimum metadata to capture
- Dataset source and version
- Inclusion/exclusion criteria
- Date of retrieval/curation
- Processing steps
- Schema or field definitions
- Model version and training configuration
- Evaluation task definitions and metrics

## 4. FAIR and reuse strategy

The team will make data, software, and models as FAIR as practicable, subject to legal, ethical, security, data-use, and partner constraints. Reusable outputs will include clear metadata, provenance, documentation, and version identifiers.
